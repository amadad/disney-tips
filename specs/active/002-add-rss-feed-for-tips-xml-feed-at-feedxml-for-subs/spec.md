# Spec: RSS Feed Discovery & Frontend Integration

## Overview

Make the existing RSS feed at `/feed.xml` discoverable by both feed readers (via HTML meta tag) and human visitors (via visible footer link).

## Requirements

### R1: Auto-Discovery Tag

**Given** a user visits `https://disney.bound.tips/` with a feed-reader-enabled browser or extension  
**When** the browser parses the HTML `<head>`  
**Then** it finds a `<link rel="alternate" type="application/rss+xml" title="Disney Tips RSS Feed" href="/feed.xml">` tag and offers feed subscription

### R2: Visible RSS Link in Footer

**Given** a user scrolls to the bottom of the page  
**When** the footer renders  
**Then** an RSS icon/link is visible that:
- Links to `/feed.xml`
- Opens in a new tab (`target="_blank"`)
- Has accessible text (aria-label or visible label "RSS Feed")
- Uses the standard RSS orange icon or a text label

### R3: Link Correctness

**Given** a user clicks the RSS footer link  
**When** the browser navigates to `/feed.xml`  
**Then** the browser displays or downloads valid RSS 2.0 XML (the existing feed generated by the pipeline)

### R4: Build Integrity

**Given** `npm run build` is executed  
**When** Vite builds the site  
**Then** `dist/index.html` contains the `<link rel="alternate">` tag and `dist/feed.xml` exists (unchanged from current behavior)
